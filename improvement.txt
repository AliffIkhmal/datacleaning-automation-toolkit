Must-Have Considerations

Data privacy/compliance: Treat HR CSVs as sensitive PII; enforce encryption in transit (HTTPS) and at rest, plus retention/deletion rules (PDPA/GDPR/local law).
Access control: Add login + role-based access (HR/Admin only); never expose upload/download endpoints publicly without auth.
Auditability: Log who uploaded, when cleaning ran, what rules changed, and what records were modified (without logging full sensitive data).
Data quality governance: Keep cleaning rules configurable (gender/job-title mappings, date formats) and versioned so results are consistent and explainable.
No risky inference: Do not infer gender from names; unresolved values should remain Unknown unless a trusted source provides it.
Technical Deployment

Environment reproducibility: Pin dependency versions and deploy with a container (Docker) or managed Streamlit host.
Runtime controls: Set upload size limits, timeouts, memory limits, and queueing for concurrent users.
Storage strategy: Use temporary secure storage for uploads and auto-delete processed files after download.
Monitoring: Add health checks, error tracking, and usage metrics (failed parses, missing columns, parse-rate by date format).
Backup/recovery: Back up configs and rule mappings; prepare rollback plan for bad releases.

Model/Data Quality Improvements (Imputation + Schema Matching)

1) Improve imputation strategy to reduce signal loss
- Add missingness flags per key field (e.g., Age_missing, Salary_missing) before filling values.
- Use segmented imputation (by Department/Job_Title/Gender group) instead of global median/mode.
- For numeric fields, prefer robust methods (KNN/MICE/regression) where data volume supports it; keep median as fallback.
- For categorical fields, consider probabilistic imputation or "Unknown" category when confidence is low.
- Track before/after distribution checks (variance, quantiles, class proportions) and alert if drift exceeds threshold.
- Keep original raw columns (or a snapshot) for traceability in downstream modeling.

2) Make schema resolution safer (reduce ambiguous alias mapping)
- Replace free-form alias matching with a versioned data dictionary (approved column names + aliases).
- Add confidence scoring for column mapping; auto-map only when confidence is high, otherwise require manual confirmation.
- Detect collisions (one source column matching multiple canonical fields) and block pipeline until resolved.
- Validate expected data type/range per mapped field (e.g., Age numeric 15-80, Salary positive) before accepting mapping.
- Log mapping decisions and expose them in the cleaning report for auditability.
- Add unit tests with intentionally ambiguous headers to prevent regression.

3) Governance and controls
- Add configurable policy in strict mode:
  - Imputation policy (global/segmented/model-based, allowed columns).
  - Mapping policy (minimum confidence threshold, manual review trigger).
- Publish data-quality KPIs in report: imputation rate by column, unresolved mappings count, and downstream drift metrics.
